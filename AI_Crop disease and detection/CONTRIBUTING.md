# Contributing to CropGuard AI

Thank you for your interest in contributing to the AI-Powered Crop Disease Detection System!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit: `git commit -m "Add your feature"`
6. Push: `git push origin feature/your-feature-name`
7. Create a Pull Request

## Code Style

### Frontend (React)
- Use functional components with hooks
- Follow React best practices
- Use TailwindCSS for styling
- Keep components small and focused

### Backend (Node.js)
- Use async/await for async operations
- Follow RESTful API conventions
- Add error handling
- Use meaningful variable names

### AI Service (Python)
- Follow PEP 8 style guide
- Add docstrings to functions
- Use type hints where possible
- Keep functions focused and small

## Testing

Before submitting a PR:
- Test your changes locally
- Ensure all services start correctly
- Test the feature you added
- Check for console errors

## Pull Request Process

1. Update README.md if needed
2. Add comments to complex code
3. Ensure your code follows the project style
4. Test thoroughly
5. Submit PR with clear description

## Questions?

Open an issue for any questions or suggestions!
